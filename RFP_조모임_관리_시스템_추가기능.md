## 1. 추가 기능 1: 카카오 소셜 로그인 연동

### 1.1 개요
사용자가 기존 계정을 카카오 계정과 연동하고, 이후 카카오 로그인을 통해 간편하게 로그인할 수 있는 기능을 제공합니다.

### 1.2 기능 요구사항

#### 1.2.1 계정 연동 기능
- **위치**: 마이페이지 또는 계정 설정 페이지
- **기능명**: "카카오 계정 연동"
- **동작 방식**:
  1. 사용자가 "카카오 계정 연동" 버튼 클릭
  2. 카카오 로그인 인증 화면 표시
  3. 사용자가 카카오 계정으로 로그인 및 동의
  4. 기존 계정 정보와 카카오 계정 정보를 데이터베이스에 연동 저장
  5. 연동 완료 메시지 표시

- **저장 정보**:
  - 카카오 사용자 고유 ID (Kakao User ID)
  - 카카오 프로필 사진 URL (선택적)
  - 카카오 닉네임 (선택적)
  - 연동 일시 (Timestamp)

- **에러 처리**:
  - 연동 실패 시 사용자에게 명확한 오류 메시지 표시
  - 이미 연동된 계정인 경우 중복 연동 방지 및 안내 메시지

#### 1.2.2 간편 로그인 기능
- **위치**: 로그인 페이지
- **UI 요소**: "카카오로 로그인하기" 버튼
- **디자인 요구사항**:
  - 카카오 브랜드 컬러 (#FEE500) 사용
  - 카카오 로고 아이콘 포함
  - 기존 로그인 버튼과 시각적으로 구분되도록 배치

- **동작 방식**:
  1. 사용자가 "카카오로 로그인하기" 버튼 클릭
  2. 카카오 로그인 인증 화면 표시 (이미 연동된 경우 자동 인증 가능)
  3. 인증 성공 시 카카오 User ID로 기존 연동된 계정 조회
  4. 연동된 계정이 존재하면 해당 계정으로 자동 로그인
  5. 연동된 계정이 없으면 "계정을 먼저 연동해주세요" 안내 메시지 표시

- **보안 요구사항**:
  - 카카오 OAuth 2.0 인증 프로토콜 준수
  - Access Token 및 Refresh Token 안전하게 관리
  - 세션 관리 및 자동 로그아웃 정책 적용

#### 1.2.3 연동 해제 기능 (선택사항)
- **위치**: 마이페이지 또는 계정 설정 페이지
- **기능**: "카카오 계정 연동 해제" 버튼
- **동작**: 연동 정보 삭제 후 일반 로그인만 가능하도록 변경

### 1.3 기술 요구사항

#### 1.3.1 카카오 개발자 콘솔 설정
- 카카오 앱 등록 및 JavaScript 키 발급
- Redirect URI 등록
- 동의 항목 설정 (필수: 카카오계정(이메일), 선택: 닉네임, 프로필 사진)

#### 1.3.2 데이터베이스 스키마
```sql
-- 사용자 테이블에 추가할 컬럼 (예시)
ALTER TABLE users ADD COLUMN kakao_user_id VARCHAR(255) UNIQUE;
ALTER TABLE users ADD COLUMN kakao_profile_image_url TEXT;
ALTER TABLE users ADD COLUMN kakao_nickname VARCHAR(100);
ALTER TABLE users ADD COLUMN kakao_linked_at TIMESTAMP;
```

#### 1.3.3 API 엔드포인트 (예시)
- `POST /api/auth/kakao/link` - 카카오 계정 연동
- `POST /api/auth/kakao/login` - 카카오 로그인
- `DELETE /api/auth/kakao/unlink` - 카카오 계정 연동 해제

### 1.4 UI/UX 요구사항
- 연동 상태 표시: 마이페이지에 "카카오 계정 연동됨" 배지 또는 아이콘 표시
- 로딩 상태: 연동 및 로그인 과정 중 로딩 스피너 표시
- 피드백: 성공/실패 시 토스트 메시지 또는 알림 표시

## 2. 추가 기능 2: 관리자 회원 관리 패널 반응형 UI/UX 개선

### 2.1 개요
관리자 패널의 회원 목록을 모바일 환경에서 가독성 있게 개선하기 위해 반응형 디자인을 적용합니다.

### 2.2 화면 분기 기준
- **Breakpoint**: `768px`
- **Wide View (데스크톱/태블릿)**: 화면 너비 ≥ 768px
  - 기존 레이아웃 유지 (모든 정보 표시)
- **Small View (모바일)**: 화면 너비 < 768px
  - 새로운 목록/상세 분리 레이아웃 적용

### 2.3 Wide View (≥ 768px) 요구사항

#### 2.3.1 레이아웃
- **유지 사항**: 현재의 모든 정보가 표시되는 테이블 형태 레이아웃 그대로 유지
- **표시 항목** (기존과 동일):
  - 프로필 사진
  - 이름
  - 학번
  - 성별
  - 학년
  - 단과대
  - 학과
  - 전화번호
  - 관리 기능 버튼 (잔류, 수정, 삭제, 관리자 추가 등)

### 2.4 Small View (< 768px) 요구사항

#### 2.4.1 회원 목록 뷰 (List View)

**목적**: 사용자를 빠르게 식별하고 스크롤하기 좋게 만듭니다.

**레이아웃**:
- 각 회원을 한 줄로 표시
- 카드 형태 또는 리스트 아이템 형태
- 터치하기 쉬운 최소 높이: 60px 이상

**표시 항목 (왼쪽부터 순서대로)**:
1. **프로필 사진** (왼쪽 정렬)
   - 크기: 40px × 40px (원형)
   - 카카오 연동 시: 카카오 프로필 사진 표시
   - 미연동 시: 기본 아바타 아이콘 표시
   - 여백: 오른쪽 12px

2. **이름** (프로필 사진 오른쪽)
   - 폰트 크기: 16px
   - 폰트 굵기: 600 (Semi-bold)
   - 색상: #1F2937 (Gray-800)

3. **학번** (이름 아래 또는 오른쪽)
   - 폰트 크기: 14px
   - 폰트 굵기: 400 (Regular)
   - 색상: #6B7280 (Gray-500)

**숨김 항목** (목록 뷰에서는 표시하지 않음):
- 성별
- 학년
- 단과대
- 학과
- 전화번호
- 관리 기능 버튼

**인터랙션**:
- 각 회원 항목은 클릭 가능 영역으로 표시 (호버 효과 또는 터치 피드백)
- 클릭 시 상세 정보 뷰로 전환

**스타일 가이드**:
- 배경색: #FFFFFF
- 구분선: 각 항목 사이 1px #E5E7EB (Gray-200)
- 패딩: 좌우 16px, 상하 12px
- 그림자: 없음 (플랫 디자인)

#### 2.4.2 상세 정보 뷰 (Detail View)

**트리거**: 회원 목록에서 특정 회원 클릭/탭

**표시 방식**: 모달(Modal) 또는 바텀 시트(Bottom Sheet) 형태

**모달 스타일**:
- 전체 화면 오버레이 또는 하단에서 올라오는 시트 형태
- 배경: 반투명 오버레이 (#000000, opacity: 0.5)
- 모달 배경: #FFFFFF
- 모달 최대 높이: 화면 높이의 80% (또는 내용에 따라 자동)

**표시 정보 (상단부터 순서대로)**:

1. **헤더 섹션**
   - 프로필 사진 (큰 사이즈: 80px × 80px)
   - 이름 (큰 폰트: 20px, Bold)
   - 닫기 버튼 (X 아이콘, 우측 상단)

2. **기본 정보 섹션**
   - 제목: "기본 정보" (섹션 헤더)
   - 정보 항목 (라벨:값 형태로 표시):
     - 이름: [이름]
     - 학번: [학번]
     - 성별: [성별]
     - 학년: [학년]
     - 단과대: [단과대]
     - 학과: [학과]
     - 전화번호: [전화번호]

3. **관리 기능 섹션** (하단 고정)
   - 버튼 레이아웃: 세로 배치 또는 그리드 형태
   - 버튼 목록:
     - "잔류" 버튼
     - "수정" 버튼
     - "삭제" 버튼
     - "관리자 추가" 버튼
   - 버튼 스타일:
     - 배경색: 각 기능에 맞는 색상 (예: 삭제는 빨간색)
     - 높이: 44px (터치하기 쉬운 크기)
     - 폰트 크기: 16px
     - 모서리: 8px (rounded)

**인터랙션**:
- 모달 외부 클릭 또는 닫기 버튼 클릭 시 모달 닫기
- 스크롤 가능: 내용이 많을 경우 스크롤 가능
- 관리 기능 버튼 클릭 시 해당 기능 실행 후 모달 닫기 또는 상태 업데이트

**애니메이션**:
- 모달 열기: 하단에서 올라오는 슬라이드 애니메이션 (300ms)
- 모달 닫기: 아래로 내려가는 슬라이드 애니메이션 (300ms)

### 2.5 기술 요구사항

#### 2.5.1 반응형 구현 방법
- CSS Media Query 사용: `@media (max-width: 767px)`
- 또는 JavaScript로 viewport width 감지하여 클래스 토글

#### 2.5.2 상태 관리
- 현재 선택된 회원 정보 상태 관리
- 모달 열림/닫힘 상태 관리

#### 2.5.3 접근성
- 키보드 네비게이션 지원 (ESC 키로 모달 닫기)
- 스크린 리더 지원 (ARIA 라벨)
- 터치 타겟 최소 크기: 44px × 44px

### 2.6 UI/UX 가이드라인

#### 2.6.1 색상 팔레트
- 배경: #FFFFFF
- 텍스트 기본: #1F2937
- 텍스트 보조: #6B7280
- 구분선: #E5E7EB
- 주요 액션: #3B82F6 (Blue-500)
- 위험 액션 (삭제): #EF4444 (Red-500)
- 성공 액션: #10B981 (Green-500)

#### 2.6.2 타이포그래피
- 제목: 20px, Bold
- 본문: 16px, Regular
- 보조 텍스트: 14px, Regular
- 라벨: 14px, Medium

#### 2.6.3 간격 (Spacing)
- 섹션 간격: 24px
- 항목 간격: 16px
- 패딩: 16px
